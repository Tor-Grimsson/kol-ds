Design

The design language of Contract IQ — a small insight for everyone who reads this far. All tokens are solid hex values, no opacity-based colours.

These are the raw values every semantic token references. Build your Figma primitives from these — one collection called "Primitives", no modes needed. Semantic tokens then reference these in a separate "Tokens" collection with Light/Dark modes.

Light mode uses cast shadows. Dark mode flips to inner highlights — panels catch light from above rather than casting shade below. Toggle dark mode to see the difference.

07 — Figma setup

How to build this in Figma Variables

Two collections, a clear hierarchy. Primitives never have modes; Tokens always do.

Collection 1 — Primitives
No modes. Raw values only. Groups: iris/, midnight/, neutral/, teal/, amber/, red/
iris/50 → #F0EFFE
iris/600 → #5A50C8
neutral/50 → #F8F8FA
neutral/600 → #6C6C76
teal/500 → #1A8055
...and so on for each scale
Collection 2 — Tokens
Two modes: Light and Dark. References primitives.
surface/canvas
· Light → {iris/50} ← wait, neutral/50
· Dark → {neutral/950}

text/primary
· Light → {neutral/950}
· Dark → {neutral/50}

border/default
· Light → #E0E0E4
· Dark → #38383E
